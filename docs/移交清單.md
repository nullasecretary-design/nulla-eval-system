# NULLA 評核系統 — 移交清單

> 給老闆 + 未來接手者(下任秘書 / 開發者)的快速上手手冊。
> 系統由 Becca 從 2026-04 設計到 2026-05 完成,正式上線 2026-05-20 前後。

## 一、系統長相

NULLA 評核系統是公司內部用的網頁工具,網址:

> **https://nulla-eval-system.vercel.app**

員工用 LINE 帳號登入,每月填一次評核。三層評核制:

| 評核者 | 占比(一般員工) | 占比(直屬執行長者) |
|--------|---------------|--------------------|
| 員工自評 | 0%(必填但只作參考) | 0%(必填但只作參考) |
| 主管評 | 70% | — |
| 執行長評 | 30% | 100% |

季度結算:每三個月一次,在 3 / 6 / 9 / 12 月,影響下一季獎金。

## 三、上線後例行操作

### 3.1 每月 1 號(系統自動做,不用人介入)

系統 cron 自動建本月「待啟動」評核紀錄。秘書當天登入後台就會看到。

### 3.2 每月 20 號以後(秘書手動做)

1. 秘書登入 → 進「評核管理」(`/admin/evaluations`)
2. 看到「啟動本月評核」表單 → 選截止日(預設本月最後一天 23:59)→ 按啟動
3. 系統會**自動**寄 Email + LINE 推播給所有員工通知開始評核

### 3.3 評核期間日常(秘書)

- 進「評核管理」看誰還沒填
- 可以「一鍵催繳全員(不含執行長)」或「單獨提醒某人」
- 員工填完自評後,系統會**自動**通知該員工的主管「輪到你了」
- 提醒執行長必須另外按「提醒執行長」按鈕(避免騷擾老闆)

### 3.4 月底截止(系統自動做)

截止時間一到,系統會自動把該月評核期關閉,沒填的標為「逾期未填」。

### 3.5 季度結算(會計 / 老闆 / 超管)

- 進「季度報表」(`/admin/reports/[年]/[季度]`)
- 看主表 + 每位員工的詳細頁
- 可下載 CSV(含下載時間 metadata,適合直接交給薪資系統)

### 3.6 員工管理(秘書)

- 新進員工:「員工管理」→「+ 新增員工」
- 大量新進(例如新診所開幕):「員工管理」→「Excel 匯入」
- 離職員工:編輯該員工 → 狀態改「已停用」(資料保留,只是不能登入)
- 員工編號 NULLA0001 ~ NULLA9999 格式

### 3.7 歷史評核分數匯入(秘書 / 會計)

如果之前是紙本評核要轉系統,或新公司剛上線要補歷史資料:

1. 「歷史匯入」(`/admin/import-history`)
2. 選季度 → 下載範本(系統幫填好年/月/姓名/角色)
3. 填上分數 → 上傳 → 預覽 → 確認

## 四、常見問題

### 4.1 員工換手機 / LINE 帳號被盜

1. 秘書進「員工管理」→ 找該員工 → 點「編輯」
2. 編輯頁底部「LINE 綁定狀態」→ 點「解除 LINE 綁定」
3. 通知員工:下次點 LINE 登入時系統會請他輸入員工編號重新綁

> ⚠️ 重要:**會計 / 執行長 / 超管 的解綁只有超管能做**,秘書沒權限。這是 spec §4.4 規定,避免有人冒充「我換手機了」騙秘書解綁高權限帳號。

### 4.2 員工要修改已送出的分數

員工**不能**自行修改。需要:

1. 秘書 / 超管 進「評核管理」找該筆 → 點解鎖
2. 系統會自動寄一封信給所有秘書 + 超管當動作備份(留下「誰解了什麼」的紀錄)
3. 通知員工:下次登入這筆會變回「待填」,可以重新送出

### 4.3 員工沒收到 LINE 推播

最常見原因:**沒加 bot `@315kvthv` 為好友**。請員工加好友後再試。

如果該員工有設 `company_email`,系統也會同時寄 Email,可以從信箱收到。

### 4.4 系統壞掉 / 網頁看不到

1. 先看 Vercel:https://vercel.com → 專案 → Deployments
   - 如果最新一筆是紅色 **Error**:代表程式部署失敗,找開發者
   - 如果是綠色 **Ready** 但網頁壞:可能是 Supabase 服務問題
2. 看 Supabase:https://supabase.com/dashboard → 專案狀態應該是 Active

### 4.5 環境變數(密碼 / 金鑰)要修改

進 Vercel → 專案 → Settings → Environments → Production → Environment Variables

完整環境變數清單見本文末「附錄 A」。

## 七、還沒做的東西(未來開發 backlog)

如果接手者要繼續開發,以下功能規格書有但**還沒實作**:

1. **多公司支援 + 切換單位 UI**(規格 §2.3 / §8.6)— 目前系統設計上支援多公司,但 UI 只給 NULLA。如果診所 1 / 2 要上線,要做「右上角切換單位」按鈕。估 2-3 小時。

2. **管理者身分變更通知**(規格 §9.1)— 改別人權限時自動寄 email 通知超管 + 會計。目前只有解鎖 / 解綁有寄 audit email,角色變更還沒做。

3. **季度報表自動寄送**(規格 §8.4)— 目前報表是手動進系統下載。未來可加自動每季初寄 CSV 給會計 + 執行長。

4. **員工自助 LINE 重綁**(規格 §4.4 path A)— 目前重綁只有後台代解綁,沒做員工自助 Email 驗證碼流程。小公司用不到,大公司可以考慮。

5. **報表「主管 / 執行長落差」欄**(規格 §8.3)— 落差 ≥ 10 分黃底警示。Becca 認為小公司不需要,故意不做。

詳細完整 backlog 見 `docs/spec_v0.3.md` 第十三節。

## 八、技術細節(給工程師看的)

### 8.1 技術選型

- 前端 + 後端:**Next.js 16**(App Router)+ React 19 + Tailwind CSS 4
- 資料庫:**Supabase**(PostgreSQL + RLS)
- 部署:**Vercel**(Hobby plan)
- 自動排程:**Vercel Cron**(在 `vercel.json` 宣告)
- 寄信:**Nodemailer + Gmail SMTP**(App Password)
- LINE 推播:**LINE Messaging API**
- 身分驗證:**LINE Login OAuth + 員工編號綁定**(無密碼)

### 8.2 程式碼結構

```
src/app/                  # Next.js App Router 路由
├── login/                # 登入相關頁
├── evaluations/me/       # 員工自己的評核頁(三合一)
├── history/              # 員工歷史紀錄(三層展開)
├── admin/                # 後台
│   ├── evaluations/      # 評核管理
│   ├── employees/        # 員工管理
│   ├── reports/          # 季度報表
│   ├── unlocks/          # 解鎖紀錄
│   └── import-history/   # 歷史評核匯入
└── api/                  # API routes
    ├── auth/             # LINE OAuth + 員工編號綁定
    ├── eval-periods/     # 啟動評核
    ├── evaluations/      # 送出 / 解鎖
    ├── admin/            # 後台 API
    └── cron/             # Vercel Cron 觸發點

src/lib/                  # 共用模組
├── session.ts            # 簽 cookie / 讀 session
├── supabase.ts           # 給瀏覽器用的 Supabase client
├── supabase-admin.ts     # 給 server 用的 admin client(繞過 RLS)
├── email.ts              # Gmail SMTP + 信件模板
└── line.ts               # LINE Messaging push + 訊息模板

supabase/
├── migrations/           # SQL DDL(schema + RLS)
├── queries/              # dev 用 SQL(清空、種測試資料)
└── seed.sql              # 初始種子資料

docs/
├── spec_v0.3.md          # 完整規格書(Becca 的設計記憶)

└── 移交清單.md            # 本文件
```

### 8.3 自動排程(Vercel Cron)

兩條 cron job(設定在 `vercel.json`):

| Cron path | Schedule (UTC) | 對應台北時間 | 用途 |
|-----------|----------------|--------------|------|
| `/api/cron/start-month` | `0 0 1 * *` | 每月 1 號 08:00 | 建本月「待啟動」row |
| `/api/cron/sweep-deadlines` | `0 0 * * *` | 每天 08:00 | 關閉到期 period + 標逾期未填 |

兩條都用 `CRON_SECRET` 環境變數 + `Authorization: Bearer ...` header 認證。

### 8.4 部署流程

GitHub `main` branch 有新 commit → Vercel **自動**觸發部署。

開發者改完程式碼:
```bash
git push origin main
```
1-2 分鐘後 Vercel Dashboard → Deployments 會看到綠色 Ready。

### 8.5 看 Production log

Vercel → 專案 → Logs(左側選單)。每個 API 呼叫都有 log。可以用「Function name」或「path」過濾。
